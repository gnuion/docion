Install custom image on Contabo

November 26, 2022 · 3 min read
Jon Zuka
Blog owner

How to install a custom Arch Image on Contabo.

Choose Custom installation​

Disable cloud-init

Choose custom install

Now use a VNC client with the provided credentials from contabo. Then choose your image. Here I chose Arch Linux.

Do a ping test:

ping google.com

Create Partitions​

Create an empty GPT partition table. Use m for a list of avalable commands.

fdisk /dev/sda

Create a bios_grub partiton of 1MiB starting at 1MiB from the start of the disk.

n
2048
+1MiB

Set type as bios_grub.

t
4

Now create the root partition with the size of 30GiB:

n
2
4096
+30GiB

Create other partitions you might need, for example home.

Format partitions with a ext4 filesystem:

mkfs -t ext4 /dev/sda2

Mount root partition in /mnt directory. In case you craeted a home partition, mount it to /mnt/home

mount /dev/sda2 /mnt
mount -o x-mount.mkdir /dev/sda4 /mnt/home

Install base system:

pacstrap -i /mnt base linux linux-firmware

Configure fstab for the new installation.

genfstab -U /mnt >> /mnt/etc/fstab

Chroot​

Change root into the new system. Then install a text editor.

arch-chroot /mnt
pacman -S nano

Set timezone:

ln -sf /usr/share/zoneinfo/Region/City /etc/localtime

Edit /etc/locale.gen by uncommenting en_US.UTF-8 UTF-8. Generate the locales.

nano /etc/locale.gen
locale-gen

Create a locale.conf file, and set the LANG variable accordingly.

# /etc/locale.conf
LANG=en_US.UTF-8

Important! Don't forget to setup a root password!

passwd

Install bootloader.

pacman -S grub
grub-install /dev/sda --target=i386-pc
grub-mkconfig -o /boot/grub/grub.cfg

Reboot and good luck 🤞.

Configuring network​

To use systemd-networkd, enable the service. When reconiguring, restart service.

systemctl enable --now systemd-networkd

Check network device name using ip a.

Wired adapter using DHCP​

# /etc/systemd/network/20-wired.network
[Match]
Name=YOUR_NETWORK_DEVICE_NAME

[Network]
DHCP=yes
DNS=8.8.8.8

Wired adapter using a static IP​

# /etc/systemd/network/20-wired.network
[Match]
Name=YOUR_NETWORK_DEVICE_NAME

[Network]
Address=YOUR_STATIC_IP_ADDRESS
Gateway=YOUR_GATEWAY
DNS=8.8.8.8
systemctl enable --now systemd-networkd

Install sudo​

pacman -S sudo

Create a new user.

useradd --create-home YOUR_USERNAME
usermod --append --groups wheel

Edit sudoes file to uncommen wheel user.

EDITOR=nano visudo

# Then find and uncomment
%wheel ALL=(ALL:ALL) NOPASSWD:ALL

# Then login to user and test
su YOUR_USERNAME
sudo echo hello

Enable SSH​

sudo pacman -S openssh
sudo systemctl enable --now sshd

Generate SSH keys on client and server.

ssh-keygen -t ed25519

Copy the content of the public key of the client and paste it to the file ~/.ssh/authorized_keys on the server.

Force public key authentication

# /etc/ssh/sshd_config
PasswordAuthentication no
AuthenticationMethods publickey
